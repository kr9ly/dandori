# ダイジェスト — なぜ効くか、工程ごとに一行ずつ

各工程が有効な理由の最小言語化。ワーキングメモリに収まるサイズを優先し、
詳細は各 SKILL.md と [philosophy.md](philosophy.md) に委ねる。
2026-07-18 の対話較正（通説的な言語化を実際のスキル定義と突合）の蒸留。

貫通原理は **主張の検証状態の会計 — 簿外の禁止**（philosophy §2）。以下は全部その各論。

1. **what 保護**（spec / impl） — what/how 分離というより一方通行弁。what の変更は
   ユーザー裁定のみ、how 側のエージェントには書き込み経路なし（時に読み取りすらなし）
2. **空間の閉包**（状態モデル） — 軸×ペアの全数裁定（検査 3）で「書いていない部分」の
   沈黙を禁止。もぐらは個体でなくモデルに還流され、同族の穴が全部再列挙される
3. **ground** — 調査でなく前提の会計。実行検証済 / 読解のみの等級づけ、
   残余はリスク格付けで spike へ
4. **spike** — 「やる人とやらない人がいる」の人間分散を機械判定に置換
   （スキップは根拠つき判定であり省略ではない）。逆行 = 手戻り防止の成功
5. **codereview** — recall はレーン、precision は反証に分離。下流で回復できる誤り
   （偽陽性）だけを許し、回復できない誤り（見落とし）を上流で禁じる
6. **plan** — 敵は飽和でなく希釈。固定するのは手順でなくスコープ・文脈・検証の3点
7. **refine** — 接地なき好みは棄却、1ラウンド打ち切り（客観的な決着装置がないから）。
   マニフェスト隔離の代償（再発明）を再発明レーンが回収する補償弁
8. **gate** — 上流の自己申告もランナーの緑も信用しない。保証は品質水準でなく
   「仕様と実装の乖離が不可視のまま存在できない」こと
9. **annotate / strip / cleanup** — Why の行き先を読者動線（pull / push）で全数確定して
   からプロセス語彙を除去（strip — 外部レビューは痕跡ゼロのコードで受ける）。文書の処分
   （cleanup）は改訂終了の裁定後のみ。Why not コメントは将来レビューの再演防止装置
10. **survey / doctor** — 依存の否定でなく依存の会計。欠けたら壊れず名指しで劣化し、
    promote で回すほど足場が複利で厚くなる
