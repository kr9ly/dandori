# dandori

## 公開リポジトリ — 実プロジェクト語彙の持ち込み禁止

このリポジトリは GitHub で公開されている。実プロジェクトでの適用結果・実験記録・
実例を反映するとき、**そのプロジェクトの固有語彙をそのまま書かない**こと。

- **仮名化の対象**: プロジェクト名、社内・外部システム名、決済/物流等のサービス名、
  キャリア名、製品名、コード上の識別子（モデル名・カラム名・ステータス値・API enum 値）
- **残してよいもの**: 汎用の機能名（cart-payment 等）、ドメイン一般語、
  汎用サービス名（Slack 等）、フレームワークに依存しない一般概念
- **仮名は構造保存**で付ける（役割が読み取れる汎用名に置き換える。意味を潰さない）

### 正準仮名語彙（既存記録と統一すること）

| 役割 | 仮名 |
|------|------|
| 適用先プロジェクト | ec-replace / EC |
| 基幹在庫システム | ERP / erp |
| 倉庫・出庫システム | WMS / wms |
| カード決済サービス | CardPay / card |
| 外部ウォレット決済 | WalletPay / wallet_pay |
| 配送キャリア | キャリアX / キャリアY（carrier_x / carrier_y） |
| 製品ライン | Model X / modelx |
| 移行元の旧システム | 旧実装 |

### コミット前チェック

実プロジェクト由来の内容を追加したら、コミット前に固有語彙が混入していないか
grep で確認する。対応表（実語彙→仮名）はこのリポジトリに**書かない**
（公開されると仮名化が無意味になる）— 必要なら NOTES.local.md（gitignore 済み）へ。
